# 02 · UI — Icon & Components (Button, Hud, Tray, FAB, ComboPopup, Dialog)

Hạng mục **UI**: bộ component vỏ (Compose) khớp `03-components/`. Đây là các phần "không phải Canvas gameplay" — menu, HUD, tray, dialog. Dựng đúng để bố cục (file 03) và Hub (file 04) ráp lại.

**Design nguồn (mỗi component có `.prompt.md` + `.jsx` + `.card.html`):**
- `02-foundations/03-icon/` — Icon set (Lucide-style, stroke 2dp).
- `03-components/01-button` · `02-hud` · `03-tray` · `04-gravity-rotate-button` · `05-combo-popup` · `06-dialog`.
- Token motion: `01-tokens/05-motion.css` (fast 150 / base 250 / medium 350 / slow 450; ease-out, ease-jelly…).

**Nguyên tắc OO:** mỗi component là một `@Composable` có state-holder/param rõ ràng, tách "vẽ candy" (edge 3D, gloss) thành modifier/shape tái dùng. Press-state, badge, ripple riêng — không nhồi vào màn hình.

**Lưu ý:** Tray và GravityRotateButton sẽ phục vụ gameplay (file 06) và hiệu ứng (file 05); ở đây dựng phần **hình + tương tác cơ bản**, chưa cần nối logic Core.

## Prompt 2 — `Button`: nút candy 3D nén khi nhấn

```
Mục tiêu: Button khớp 03-components/01-button (component được dùng nhiều nhất).

Đọc trước: 01-button/Button.prompt.md, Button.jsx, button.card.html; token màu (Primary/Gravity/Success/Danger…) + motion (t-tap = fast/ease-out).

Việc:
- Button(variant, size, icon, iconRight, comingSoon, fullWidth, enabled, onClick, content).
- variant: primary | gravity | success | danger | secondary | ghost — màu từ token tương ứng (primary=tangerine, gravity=purple…).
- size: cta (56dp, radius 28) | md (48dp). Luôn ≥ 48dp touch.
- Hiệu ứng candy: dưới nút có "edge" tô đặc màu edge (under-shadow 0 Npx 0 edge) tạo độ dày 3D; khi nhấn, mặt nút translateY xuống bằng độ dày edge (nén) trong motion-fast/ease-out.
- comingSoon: làm mờ (opacity ~0.55) + gắn pill "SẮP CÓ" + disable.
- icon/iconRight: dùng Icon từ Prompt 1.

Acceptance: 6 variant + 2 size + comingSoon khớp button.card.html; nhấn thấy nén xuống đúng độ sâu edge; disabled mờ.
Kiểm chứng: @Preview ma trận variant×size + comingSoon; quay/ị mô tả trạng thái nhấn.
```

## Prompt 4 — `Tray`: dock 112dp chứa 3 mảnh

```
Mục tiêu: Tray khớp 03-components/03-tray (phần hình + chọn slot; nối kéo-thả ở file 06).

Đọc trước: 03-tray/Tray.prompt.md, Tray.jsx, tray.card.html. Lưu ý catalog 24 shape (6 nhóm) và quy tắc cell-size chung.

Việc:
- Tray(pieces: List<Piece?> kích thước 3, selectedIndex, onSelect). Cao 112dp (GjDimens.trayHeight), slot 96dp.
- MỌI slot dùng CHUNG một cell-size, lấy theo mảnh lớn nhất hiện có, clamp 12–22dp — dock không lẫn ô to/nhỏ. Mảnh 1..9 ô đều vừa.
- null = slot đã tiêu thụ → placeholder gạch đứt.
- Vẽ mảnh bằng JellyBlockRenderer (file 01) — mảnh tự normalize toạ độ, bo góc đầy đủ.
- Slot được chọn: nâng 6dp + viền focus primary (theo particles-juice: lift translateY(-6) + scale 1.04, ease-jelly khi grab — ở đây làm trạng thái selected; kéo-thả thật ở file 06).

Acceptance: 3 slot cùng cell-size, mảnh to/nhỏ đều cân; placeholder đúng; selected nâng + ring; khớp tray.card.html.
Kiểm chứng: @Preview với [mảnh nhỏ, mảnh I-5, null] và selectedIndex=0/1; so card.
```
